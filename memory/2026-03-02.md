### Backend API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Health check with DB + Vision status |
| `/entries` | GET | List entries (paginated) |
| `/entries` | POST | Create entry (auto-vision if photo) |
| `/entries/today` | GET | Today's entries + totals |
| `/entries/date/:date` | GET | Entries for specific date |
| `/entries/stats/recent` | GET | Recent days summary |
| `/entries/:id` | GET/PUT/DELETE | CRUD single entry |
| `/vision/analyze` | POST | Analyze food photo |
| `/vision/status` | GET | Vision service status |
| `/barcode/lookup/:code` | GET | Look up product by barcode |
| `/barcode/status` | GET | Barcode service status |

## 📱 App Features Summary

### Core Features
- **Today's Dashboard** - Daily calories, macros, progress rings
- **Add Food** - Photo, barcode, or manual entry
- **AI Analysis** - OpenAI Vision estimates nutrition from photos
- **Barcode Scanner** - Open Food Facts product lookup
- **History** - 7-day view with bar charts
- **Settings** - Customizable goals, API status

### Technical Stack
- **Frontend:** Expo SDK 52, React Native, TypeScript
- **Backend:** Hono.js, SQLite with WAL mode
- **AI:** OpenAI Vision / OpenRouter
- **Barcode:** Open Food Facts API
- **State:** React hooks + AsyncStorage

### Screens
1. `index.tsx` - Today's dashboard
2. `add.tsx` - Add food (photo/barcode/manual)
3. `history.tsx` - 7-day history with charts
4. `settings.tsx` - Goals and API status
5. `entry/[id].tsx` - Entry details
6. `edit/[id].tsx` - Edit/delete entries
